# Scope

本文件作用于仓库根目录及其未被下层 `AGENTS.md` 重写的所有子目录。

# Rule Priority

1. 只有落盘到 `AGENTS.md` 的规则才生效。
2. 更近目录的 `AGENTS.md` 优先于上层同类规则。
3. 未被下层重写的规则继续继承。

# Change Boundaries

1. 先做最小盘点，再开始改动。
2. 只修改与当前任务直接相关的仓库内文件。
3. 不因目录已存在而默认新增模块、执行入口、注册关系或 workspace 边界。
4. 不回滚、覆盖或整理自己未引入的现有改动，除非得到明确确认。
5. 控制面文件分工保持明确：协作约束写入 `AGENTS.md`，说明与索引写入 `README.md`，稳定命令入口写入 `justfile`。

# Stop Conditions

1. 需求边界不清且存在多种互斥实现时，必须暂停并请求确认。
2. 需要新增模块、入口、注册关系、workspace 边界或执行面模式时，必须暂停并请求确认。
3. 发现项目结构、入口组织或控制面职责异常，且无法在当前任务中做最小修复时，必须暂停并说明风险。

# Human Handoff

1. 需要做破坏性操作或历史改写时，交由人工确认。
2. 无法完成最小相关验证时，必须明确说明缺口、原因与风险，再交由人工决定是否继续。

# Engineering Baseline

1. 优先保持现有架构分层，不为局部便利破坏长期一致性。
2. 避免与当前任务无关的功能扩展、顺手重构或批量风格清理。
3. 能完成最小相关验证时，必须先验证再交付。
4. 交付说明只陈述已完成并已验证的事实；未验证项、假设项与风险项必须显式区分。
5. 尽量使用中文，除非专用名称，包括但不限于文档、注释、错误和日志信息。
